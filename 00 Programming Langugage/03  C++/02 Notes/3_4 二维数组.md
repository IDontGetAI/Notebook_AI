## 核心概念
- 二维数组在内存中的存储本质是一维数组
- 物理内存中不存在多维结构，采用连续线性空间存储
- 通过不同的索引计算方式实现多维访问
- 索引转换公式：`a[i][j] = base_address + (i * col_num + j) * sizeof(type)`

```cpp
// 二维数组定义
data_type array_name[row][col];

// 访问二维数组元素
array_name[i][j]
// 等价于 *(array_name + i * col + j)

// 索引计算公式
offset = i * 列数 + j
```

```cpp
#include <iostream>
using namespace std;

int main() {
    // 定义3行2列的二维数组
    int a[3][2] = {{1, 2}, {3, 4}, {5, 6}};

    // 二维数组访问
    cout << "二维数组访问:" << endl;
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 2; j++) {
            cout << "a[" << i << "][" << j << "] = " << a[i][j] << " ";
        }
        cout << endl;
    }

    // 使用一维方式访问（证明底层是一维存储）
    cout << "\n一维方式访问:" << endl;
    int *p = (int *)a;  // 将二维数组强制转换为一维指针
    for (int i = 0; i < 6; i++) {
        cout << "p[" << i << "] = " << p[i] << " ";
    }
    cout << endl;

    // 手动计算索引访问
    cout << "\n手动计算索引访问:" << endl;
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 2; j++) {
            // a[i][j] 等价于 *(a + i*2 + j)
            int *ptr = (int *)a + i * 2 + j;
            cout << "*(a + " << i << "*2 + " << j << ") = " << *ptr << " ";
        }
        cout << endl;
    }

    return 0;
}
```

## 运行结果 / 现象


```
二维数组访问:
a[0][0] = 1 a[0][1] = 2
a[1][0] = 3 a[1][1] = 4
a[2][0] = 5 a[2][1] = 6

一维方式访问:
p[0] = 1 p[1] = 2 p[2] = 3 p[3] = 4 p[4] = 5 p[5] = 6

手动计算索引访问:
*(a + 0*2 + 0) = 1 *(a + 0*2 + 1) = 2
*(a + 1*2 + 0) = 3 *(a + 1*2 + 1) = 4
*(a + 2*2 + 0) = 5 *(a + 2*2 + 1) = 6
```

## 原理理解

- 内存存储方式：所有数组元素按顺序线性排列，物理上不存在多维结构
- 索引转换规则：
  - 一维形式：`a[6]`表示长度为6的连续存储空间
  - 二维形式：`a[3][2]`通过公式`i × 列数 + j`映射到一维空间
- 访问示例：
  - 第一行元素(a[0][0], a[0][1])对应一维索引0和1
  - 第二行元素(a[1][0], a[1][1])对应一维索引2和3
  - 第三行元素(a[2][0], a[2][1])对应一维索引4和5
- 循环索引计算：
  - i=0, j=0,1: 索引0×2+0=0, 0×2+1=1
  - i=1, j=0,1: 索引1×2+0=2, 1×2+1=3
  - i=2, j=0,1: 索引2×2+0=4, 2×2+1=5
- 关键结论：二维访问本质是一维索引的算术转换，公式`i×列数+j`实现逻辑分层到物理连续的映射

## 易错点

- 二维数组的行和列容易混淆

- 索引计算时注意列数不是行数

- 多维数组的内存布局需要理解清楚

- 数组名转换为一维指针时需要注意类型匹配
