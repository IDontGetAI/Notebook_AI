# 3_9 递归函数

## 核心概念

- **递归函数**：在函数内部调用自身的函数
- **终止条件（Base Case）**：递归结束的边界条件
- **递归调用（Recursive Case）**：函数调用自身的部分
- **栈帧**：每次递归调用都会在栈上创建新的函数帧

## 原理理解

递归的两大要素：

| 要素 | 说明 | 重要性 |
|------|------|--------|
| **终止条件** | 递归何时结束 | 防止无限递归 |
| **递归调用** | 如何向终止条件靠近 | 确保能够到达终止条件 |

递归的工作原理：

1. 每次函数调用都会在**调用栈**上创建新的栈帧
2. 栈帧保存局部变量、参数和返回地址
3. 达到终止条件后开始返回
4. 栈帧逐层弹出，返回结果逐层传递

## 代码示例

```cpp
#include <iostream>
using namespace std;

// 示例 1：计算阶乘 n!
int factorial(int n) {
    if (n <= 1) return 1;  // 终止条件
    return n * factorial(n - 1);  // 递归调用
}

// 示例 2：斐波那契数列
int fibonacci(int n) {
    if (n <= 1) return n;  // 终止条件
    return fibonacci(n - 1) + fibonacci(n - 2);  // 递归调用
}

// 示例 3：递归打印数字
void printNumbers(int n) {
    if (n <= 0) return;  // 终止条件
    cout << n << " ";
    printNumbers(n - 1);  // 递归调用
}

int main() {
    // 阶乘
    cout << "5! = " << factorial(5) << endl;  // 120

    // 斐波那契
    cout << "fibonacci(6) = " << fibonacci(6) << endl;  // 8

    // 递归打印
    cout << "printNumbers(5): ";
    printNumbers(5);  // 5 4 3 2 1

    return 0;
}
```

## 运行结果

```
5! = 120
fibonacci(6) = 8
printNumbers(5): 5 4 3 2 1
```

## 递归过程图解（factorial(4)）

```
factorial(4)
├── 4 * factorial(3)
│   ├── 3 * factorial(2)
│   │   ├── 2 * factorial(1)
│   │   │   └── 1 (终止条件)
│   │   └── 2 * 1 = 2
│   └── 3 * 2 = 6
└── 4 * 6 = 24
```

## 易错点

- **缺少终止条件**：导致无限递归，栈溢出
- **终止条件不正确**：可能永远无法到达终止条件
- **递归深度过大**：消耗大量栈空间，可能导致栈溢出
- **重复计算**：如斐波那契递归存在大量重复计算

## 总结

递归是函数调用自身的编程技巧，必须包含终止条件和递归调用两部分，理解调用栈有助于理解递归的执行过程。
