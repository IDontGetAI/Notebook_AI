## 核心概念
- 数组的实现原理是通过指针完成的
- 数组在内存中连续分配存储空间
- 数组名代表数组首元素的地址
- 数组访问`a[i]`本质上是`*(a + i)`的语法糖

```cpp
// 定义数组
data_type array_name[size];

// 数组元素访问
array_name[index]       // 等价于 *(array_name + index)
*(array_name + index)   // 指针运算访问
```

```cpp
#include <iostream>
using namespace std;

int main() {
    int a[5];  // 定义包含5个int类型元素的数组
    // 内存分配：连续分配5个int大小的空间，每个int占4字节，总大小20字节

    // 数组名a代表数组首元素的地址，类型为int*
    cout << "数组首地址: " << a << endl;
    cout << "&a[0]: " << &a[0] << endl;

    // 指针运算访问
    a[0] = 10;
    cout << "a[0]: " << a[0] << endl;        // 等价于 *(a + 0)
    cout << "*(a + 0): " << *(a + 0) << endl;

    // a + 1 表示指针向后移动一个int大小（4字节）
    *(a + 1) = 20;
    cout << "a[1]: " << a[1] << endl;

    return 0;
}
```

## 运行结果 / 现象

- 数组名和`&a[0]`输出相同的地址值
- `a[i]`和`*(a + i)`访问结果完全一致
- 数组元素在内存中连续存储

## 原理理解

- 数组的底层实现通过指针完成
- `a[i]`的底层实现为`*(a + i)`，即通过指针偏移访问元素
- `a + 1`表示指针向后移动一个类型大小的距离（int类型移动4字节）
- 指针移动单位由指针类型决定：
  - int* 指针+1移动4字节
  - char* 指针+1移动1字节
- 语法糖：`a[i]`等价于`*(a + i)`，是编译器提供的简化写法
- 双向移动：指针支持加减运算，可向前或向后移动

## 易错点

- 数组名是常量指针，不能进行赋值操作（如`a = a + 1`非法）

- 指针运算的偏移量由指向的数据类型大小决定

- 不要混淆数组名和普通指针的区别

- 数组越界访问可能导致未定义行为
