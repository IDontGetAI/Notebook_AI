### 一、编译器和解释器

﻿

00:01﻿

高级语言编写的代码（如C语言的.c文件）需要通过翻译转换为计算机可执行程序（如Windows的.exe文件）。翻译方法主要分为两种：

- 编译（Compiling）：将源代码整体转换为机器码文件
- 解释（Interpretation）：逐行实时翻译并执行代码

#### 1.编译过程

﻿

01:31

﻿

C语言采用编译器将源代码文件（类比中文）完整翻译为机器码文件（类比英文/二进制文件），生成的可执行文件（如.exe）可直接由计算机运行。编译的核心特征是一次性完成全部代码的转换。

#### 2.解释过程

﻿

02:37

﻿

解释型语言通过解释器（类比实时翻译官）逐句翻译并执行代码，执行流程为“翻译一句→执行一句”。与编译型语言的区别在于：

- 编译型：提前完成全部翻译，运行时直接执行机器码
- 解释型：运行时同步进行翻译与执行

#### 3.编译和解释的区别

﻿

03:46

﻿

|   |   |   |
|---|---|---|
|对比维度|编译型语言（如C++）|解释型语言（如Python）|
|兼容性|依赖目标系统编译器，跨平台性差|通过解释器适配系统，兼容性强|
|执行效率|运行速度快（预编译完成）|运行速度慢（实时翻译）|
|典型代表|C/C++、Go|Python、MATLAB|

关键结论：解释型语言通过实时翻译实现跨平台，但牺牲执行效率；编译型语言需针对不同系统重新编译。

#### 4.编译型语言和解释型语言的概念

﻿

06:55

﻿

Java属于混合型语言，其编译过程分为两步：

- 前端编译：将源代码转换为中间代码（如.class文件）
- 后端解释：通过JVM（Java虚拟机，本质是解释器）执行中间代码核心特点：兼具编译型语言的预处理优势和解释型语言的跨平台能力。
