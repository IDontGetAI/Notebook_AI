# 1_7 编译器和解释器

## 核心概念

- **编译器**：将源代码一次性编译为可执行文件
- **解释器**：逐行解释执行源代码
- C++ 使用编译器，Python 使用解释器

## 原理理解

编译器 vs 解释器对比：

| 特性 | 编译器 | 解释器 |
|------|--------|--------|
| 执行方式 | 一次性编译整个源代码 | 逐行解释并执行 |
| 输出 | 生成独立的可执行文件 | 不生成可执行文件 |
| 执行速度 | 快 | 慢 |
| 调试 | 相对困难 | 方便 |
| 代表语言 | C, C++, Rust | Python, JavaScript, Ruby |

## 易错点

- C++ 程序需要**先编译后运行**
- **编译错误**在编译阶段发现，**运行时错误**在执行时发现

## 总结

C++ 是编译型语言，需要先编译成可执行文件再运行。
