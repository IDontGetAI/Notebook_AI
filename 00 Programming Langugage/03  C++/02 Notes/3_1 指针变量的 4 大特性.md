## 核心概念
- 指针变量本质上与普通变量一样，具有变量的四大基本特征
- 指针变量存储的是内存地址，而非直接的数据值
- 指针类型决定了解引用时访问的内存空间大小
- 指针在32位系统占4字节，在64位系统占8字节

## 语法定义
```cpp
// 创建指针变量
int *p = &变量名;

// 指针变量定义
data_type *pointer_name;

// 取地址运算符
&variable_name

// 取内容（解引用）运算符
*pointer_name
```

## 示例代码

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = 10;          // 普通变量
    int *p = &a;         // 指针变量p存储a的地址

    cout << "a的值: " << a << endl;           // 输出: 10
    cout << "a的地址: " << &a << endl;        // 输出: a的内存地址
    cout << "p的值(地址): " << p << endl;      // 输出: a的内存地址
    cout << "*p的值: " << *p << endl;         // 输出: 10 (解引用)
    cout << "p的地址: " << &p << endl;        // 输出: 指针p自身的地址

    return 0;
}
```

## 运行结果 / 现象
- 内存地址通常以十六进制表示（如0x7ffe...）
- 实际地址值由系统随机分配
- 指针变量自身也有独立的内存地址

## 原理理解
- 取地址运算符`&`：获取变量的内存地址
- 取内容运算符`*`：访问指针指向的内存单元内容
- 内存模型：指针变量p存储变量a的地址，通过`*p`可访问a的值
- 关键区别：
  - 声明语句中的`*`表示指针类型（如`int *p`）
  - 表达式中的`*`作为解引用运算符使用（如`*p = 5`）

## 易错点
- 指针类型必须与指向变量的数据类型匹配（`int*`不能指向`char`）
- 声明时的`*`和运算时的`*`含义不同
- 未初始化的指针不能使用（野指针）
- 注意区分指针存储的地址和指针自身的地址

## 面试 / 考试常见问法
- `&`和`*`操作符的区别是什么？
- 为什么指针变量需要类型？
- 指针在32位和64位系统中的大小分别是多少？
- 什么是指针的解引用？

## 一句话总结
- 指针是存储内存地址的变量，通过`&`获取地址，通过`*`访问内容，类型决定了指针运算的步长和访问的内存大小
