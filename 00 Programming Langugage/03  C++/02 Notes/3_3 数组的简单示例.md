## 核心概念
- 数组名代表数组首元素的地址，但实际输出时需注意值和地址的区别
- 数组元素个数可通过sizeof计算：`sizeof(a) / sizeof(a[0])`
- 数组越界访问可能导致未定义行为或程序崩溃
- 指针大小与指向类型无关（32位系统4字节，64位系统8字节）

```cpp
// 数组初始化
int a[5] = {0};  // 全部初始化为0
int a[5] = {1, 2, 3};  // 部分初始化，剩余自动为0

// 计算数组元素个数
int n = sizeof(a) / sizeof(a[0]);

// 变量初始化（C++风格）
int b(5);  // 等价于 int b = 5;
```

```cpp
#include <iostream>
using namespace std;

int main() {
    // 示例1：数组初始化和地址输出
    int a[5] = {0};  // 全部初始化为0
    cout << "a[0] = " << a[0] << endl;

    int b(5);  // C++风格初始化，b = 5
    cout << "b = " << b << endl;

    // 打印数组地址（16进制）
    cout << "数组地址: " << a << endl;

    // 示例2：计算数组元素个数
    int n = sizeof(a) / sizeof(a[0]);
    cout << "数组元素个数: " << n << endl;
    // sizeof(a) = 20字节（5个int）
    // sizeof(a[0]) = 4字节（1个int）
    // n = 20 / 4 = 5

    // 示例3：数组越界访问
    cout << "a[5] = " << a[5] << endl;  // 越界！可能输出随机值
    // a[10000] 将导致程序崩溃（访问冲突 0xC0000005）

    return 0;
}
```

## 运行结果 / 现象

- 初始化后的数组元素值为0
- 地址以16进制显示（32位系统8位16进制，64位系统16位16进制）
- 数组越界访问可能不报错（小范围越界）或导致崩溃（大范围越界）

## 原理理解

- 16进制数特性：每位包含16种可能（0-9，A-F）
- 与二进制的对应关系：1位16进制数 = 4位二进制数（2^4=16）
- 地址长度计算：32位系统地址为8位16进制（32位二进制），64位系统为16位16进制（64位二进制）
- 数组名特殊处理：虽然数组底层通过指针实现，但`sizeof(a)`返回数组总大小而非指针大小
- 内存管理原理：程序运行时获得大块预留空间，数组仅占用其中部分区域
- 越界访问机制：小范围越界可能在预分配内存空间内，大范围越界触发访问冲突

## 易错点

- 数组名在sizeof中表现特殊：返回数组总大小而非指针大小

- 指针变量本身的大小是固定的，与指向类型无关

- 数组越界访问即使不报语法错误也是非法行为

- 64位系统下地址长度是32位系统的2倍
