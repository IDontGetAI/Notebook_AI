## 核心概念
- 函数的核心作用在于代码复用和抽象封装
- 函数参数传递的本质是创建参数的副本变量（值传递）
- 返回值原理是将函数调用处替换为返回表达式
- 函数内部变量与外部同名变量互不影响（作用域隔离）

```cpp
// 函数定义四要素

// 1. 无输入无输出
void func1() {
    // 仅执行固定操作
}

// 2. 有输入无输出
void func2(int a) {
    // 处理输入参数但不返回结果
}

// 3. 无输入有输出
int func3() {
    return 42;  // 生成固定返回值
}

// 4. 有输入有输出
int func4(int a) {
    return a * a;  // 接收参数并返回计算结果
}
```

```cpp
#include <iostream>
using namespace std;

// 示例函数
void printHello() {
    cout << "Hello" << endl;
}

void printValue(int a) {
    cout << "Value: " << a << endl;
}

int getConstant() {
    return 100;
}

int square(int a) {
    return a * a;
}

int main() {
    // 调用函数
    printHello();           // 无输入无输出
    printValue(5);          // 有输入无输出

    int x = getConstant();  // 无输入有输出
    cout << "Constant: " << x << endl;

    int y = square(4);      // 有输入有输出
    cout << "Square: " << y << endl;

    return 0;
}
```

## 运行结果 / 现象


```
Hello
Value: 5
```

## 原理理解

- 函数的优势：
  - 提升开发效率：避免重复编写相同逻辑
  - 降低理解成本：调用者无需关注函数内部实现细节
- 参数传递原理：
  - 以`func(int a)`为例，调用`func(2)`时实际发生：`int a = 2;`（在函数作用域内新建变量）
  - 创建的是参数的副本，函数内修改不影响外部变量
- 返回值原理：
  - 将函数调用处替换为返回表达式
  - `int x = func();`等价于直接赋值返回结果
- 作用域隔离性：
  - 函数内部变量与外部同名变量互不影响
  - 函数参数形成独立的作用域

## 易错点

- 隐式类型转换风险：传入参数类型需与声明匹配

- 注意值传递和引用传递的区别

- 函数声明和定义的位置关系

- 返回值类型不匹配可能导致截断或错误
