# 1_5 32 位和 64 位系统

## 核心概念

- 32 位系统最大支持 4GB 内存 (2^32)
- 64 位系统支持更大的内存空间 (2^64)
- 32 位系统指针占 4 字节，64 位系统指针占 8 字节

## 原理理解

32 位 vs 64 位系统对比：

| 特性 | 32 位系统 | 64 位系统 |
|------|-----------|-----------|
| 地址总线 | 32 位 | 64 位 |
| 最大寻址空间 | 4GB | 16EB (理论值) |
| 指针大小 | 4 字节 | 8 字节 |

## 代码示例

```cpp
// 在 64 位系统上
sizeof(int*) = 8   // 指针占 8 字节
sizeof(int)  = 4   // int 占 4 字节

// 在 32 位系统上
sizeof(int*) = 4   // 指针占 4 字节
sizeof(int)  = 4   // int 占 4 字节
```

## 易错点

- 指针大小与**系统位数**相关，与指针类型无关
- **所有类型的指针**在同一系统上大小相同

## 总结

系统位数决定了指针大小和最大内存寻址空间。
