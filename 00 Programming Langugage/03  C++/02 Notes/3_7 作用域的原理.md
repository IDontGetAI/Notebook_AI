# 3_7 作用域的原理

## 核心概念

- **作用域（Scope）**：变量在程序中可被访问的范围
- **局部变量**：在函数或代码块内定义的变量
- **全局变量**：在所有函数之外定义的变量
- **生命周期**：变量从创建到销毁的时间段

## 原理理解

C++ 中的作用域类型：

| 作用域类型 | 定义位置 | 生命周期 | 可见范围 |
|------------|----------|----------|----------|
| **局部作用域** | 函数或代码块内 | 代码块执行期间 | 当前代码块 |
| **全局作用域** | 所有函数之外 | 程序运行全程 | 整个文件 |
| **类作用域** | 类定义内 | 对象存在期间 | 类成员函数 |

作用域规则：

- **大括号 `{}` 定义一个作用域**
- 内层作用域可以访问外层作用域的变量
- 同名变量在内层作用域会遮蔽（shadow）外层变量
- 离开作用域后，局部变量自动销毁

## 代码示例

```cpp
#include <iostream>
using namespace std;

int globalVar = 100;  // 全局变量

int main() {
    int localVar = 10;  // 局部变量

    cout << "main 函数内：" << endl;
    cout << "localVar = " << localVar << endl;
    cout << "globalVar = " << globalVar << endl;

    {  // 新的作用域
        int localVar = 20;  // 遮蔽外层的 localVar
        cout << "\n内层作用域：" << endl;
        cout << "localVar = " << localVar << endl;
        cout << "globalVar = " << globalVar << endl;
    }

    cout << "\n离开内层作用域后：" << endl;
    cout << "localVar = " << localVar << endl;  // 恢复为 10

    return 0;
}
```

## 运行结果

```
main 函数内：
localVar = 10
globalVar = 100

内层作用域：
localVar = 20
globalVar = 100

离开内层作用域后：
localVar = 10
```

## 易错点

- **变量遮蔽**：内层同名变量会遮蔽外层变量，但不改变外层变量的值
- **全局变量滥用**：过多使用全局变量会降低代码可维护性
- **作用域混淆**：离开作用域后访问局部变量会导致未定义行为

## 总结

作用域决定了变量的可见性和生命周期，合理使用作用域可以避免命名冲突，提高代码的可维护性。
