# 第 3 章 指针、数组、字符串与函数基础

## 上级导航
- [[C++ 程序设计 MOC]]

## 本章内容

### [[3_1 指针变量的 4 大特性]]
指针变量存储内存地址而非直接的数据值，具有变量的四大特性。通过`&`获取变量地址，通过`*`访问指针指向的内容。指针类型决定解引用时访问的内存空间大小和指针运算的步长。32位系统指针占4字节，64位系统占8字节。声明时的`*`表示指针类型，表达式中的`*`是解引用运算符。

### [[3_2 数组的原理]]
数组在内存中连续分配存储空间，数组名代表数组首元素的地址（类型为`int*`）。数组访问`a[i]`本质是`*(a + i)`的语法糖。`a + 1`表示指针向后移动一个类型大小（int移动4字节，char移动1字节）。指针类型决定指针加减运算时的偏移量大小。

### [[3_3 数组的简单示例]]
数组初始化可用`int a[5] = {0}`将全部元素初始化为0。数组元素个数计算：`sizeof(a) / sizeof(a[0])`。数组名在sizeof中特殊处理，返回数组总大小而非指针大小。地址以16进制表示，32位系统8位16进制，64位系统16位16进制。数组越界访问可能导致未定义行为或程序崩溃。

### [[3_4 二维数组]]
二维数组在内存中的存储本质是一维数组，物理内存中采用连续线性空间存储。索引转换公式：`a[i][j]`对应一维索引`i × 列数 + j`。可通过双重循环遍历，外层循环控制行，内层循环控制列。理解二维数组的内存布局有助于掌握多维数组和指针数组的关系。

### [[3_5 字符串]]
C语言字符数组以`'\0'`作为结束符，字符数组变量名本质是指向内存地址的指针。C语言中用`==`比较字符数组比较的是地址而非内容。C++的string类通过运算符重载实现内容比较功能，使用更方便。字符数组初始化时要确保容量足够容纳字符串及结束符。相同内容的字符数组可能存储在不同的内存地址。

### [[3_6 函数]]
函数的核心作用在于代码复用和抽象封装。函数参数传递的本质是创建参数的副本变量（值传递），函数内修改不影响外部变量。返回值原理是将函数调用处替换为返回表达式。函数内部变量与外部同名变量互不影响（作用域隔离）。函数定义四要素：无输入无输出、有输入无输出、无输入有输出、有输入有输出。

### [[3_7 作用域的原理]]
作用域由大括号界定，每对大括号创建一个新的作用域层级。访问规则：内层可访问外层变量，外层不可访问内层变量。作用域关系可用树形结构表示。for/if等流程控制语句创建新的作用域层级。跨作用域访问通过函数参数传递和返回值实现数据交互。

### [[3_8 作用域的简单示例]]
局部变量优先级高于全局变量。函数参数形成新的作用域，函数内部未定义的变量自动访问外层作用域。for/if等流程控制语句共享所属函数的栈空间，仅函数调用会触发新的栈空间分配。函数栈空间在函数调用时分配，函数结束时立即销毁。main函数的栈空间生命周期等同于程序运行周期。

### [[3_9 递归函数]]
递归函数通过调用自身实现循环功能，必须设置终止条件避免无限递归。递归与循环的本质均为重复执行操作，但递归内存占用随调用次数线性增长。每次函数调用创建独立栈帧，递归深度过大会导致栈溢出（stack overflow）。递归适用于树形结构遍历（如前/中/后序遍历），循环适用于简单循环控制。
