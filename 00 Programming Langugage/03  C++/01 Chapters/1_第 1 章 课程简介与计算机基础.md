# 第 1 章 课程简介与计算机基础

## 上级导航
- [[C++ 程序设计 MOC]]

## 本章内容

### [[1_1 课程简介]]
C++编程课程概述，面向已具备C语言和Python基础的学生。重点介绍计算机专业四大核心课程：数据结构、算法分析、操作系统、计算机网络，强调编程能力培养的实质在于算法思想而非语言工具本身。

### [[1_2 编程对人工智能专业的意义]]
人工智能专业需要编程的原因在于编程是实现算法和开发AI应用的基础工具。该专业是计算机科学与数学的交叉学科，编程能力与数学理论构成两大支柱。数学课程（线性代数、概率论、离散数学）为算法设计提供理论支持，编程能力则将数学理论转化为实际应用。

### [[1_3 计算机的部件]]
计算机主要由存储设备（硬盘）、内存（Memory）、CPU、通信设备、输入/输出设备、主板总线构成。内存是编程中频繁交互的核心组件，变量存储和动态内存分配都需要程序员主动干预管理。

### [[1_4 内存的逻辑理解]]
内存可逻辑理解为长管状结构，存储的本质是二进制数据。字节是内存最小可操作单位（1 byte = 8 bit），每个字节单元有唯一的内存地址标识。不同类型数据占用字节数不同：char占1字节，int占4字节。数据类型转换的关键在于解释方式，如字符'a'实际存储为ASCII码97的二进制形式。

### [[1_5 32 位和 64 位系统]]
32位系统地址长度为4字节（内存上限约4GB），64位系统地址长度为8字节（理论内存极大）。内存地址类似于门牌号，用于标识存储位置。理解内存地址与物理存储的对应关系是掌握计算机体系结构的基础。

### [[1_6 编程语言的发展]]
从机器语言到高级语言的演进过程。了解编程语言发展史有助于理解不同语言的设计理念和适用场景。

### [[1_7 编译器和解释器]]
高级语言代码需要通过翻译转换为计算机可执行程序。编译（Compiling）将源代码整体转换为机器码文件；解释（Interpretation）逐行实时翻译并执行代码。编译型语言（C/C++）执行效率高但跨平台性差；解释型语言（Python）兼容性强但运行速度慢。Java属于混合型语言，通过JVM实现跨平台。

### [[1_8 操作系统是计算机管理员]]
操作系统负责解释并执行文件内容，是程序运行的基础环境。可执行文件的运行流程：存储于硬盘 → 操作系统接收执行指令 → 加载到内存 → CPU逐条读取执行。操作系统作为资源管理器，控制所有程序的执行权限和生命周期。

## 本章重点
- **计算机基本组成**：理解 CPU、内存、硬盘等核心部件的功能与关系
- **内存模型**：掌握内存的逻辑结构，理解地址、存储单元的概念
- **32 位与 64 位系统**：理解两种架构的区别，特别是对内存寻址和指针大小的影响
- **编程语言发展史**：了解从机器语言到高级语言的演进过程
- **编译器工作原理**：理解源代码到可执行文件的转换过程
- **操作系统的作用**：理解操作系统作为资源管理器的核心功能
